 What is Microservices Architecture ?
 Ans:- It is Architectureal style structure COLLECTION OF SMALL AUTONOMOUS SERVICES model around the BUSINESS DOMAIN
       each service SELF-CONTAINT and implemented SINGLE BUSINESS CAPABILITY
	   each microservice seprate data module and interact each other to multiple instances
 
 ex:- Ecommerce application ->  client browser -> UI microservices --> customer MS --> product MS --> cart MS (totaly dependent & losely coupled) 
 
 Features of Microservices --> 
 1. small focused --> simplicity
 2. loosely coupled --> each service independent (development & deployment real quick)
 3. language nutral (one is python and other is java) 
 4. bounded context (no need implementation of other services)
 
 Advantages of Microservices --> 
 1. independent development
 2. mix technologies stack
 3. fault isolation (if any service is break down not affected the other services)
 4. granual scalling

1. Java is pure object oriented or not?
A:- 

There are lot of arguments around whether Java is purely object oriented or not. 
Java should be considered as purely object oriented language as it has wrapper classes. 
So you can use Integer, Float etc. instead of int, float etc. (there are a total of eight primitive types).
But since Java has those eight primitive types, the critics will say Java is not purely object-oriented.


2. what is primary key and unique in sql & composite key & foreign key ?
A:-

Primary key :- A PRIMARY KEY has a UNIQUE constraint by default.
               one PRIMARY KEY constraint per table.
               PRIMARY key not allows NULL 
			   Clustered Index on primary-key 
			
Unique Key:-  UNIQUE key allows only one NULL values.
              many UNIQUE constraints per table
              Clustered Index on primary-key if not exists and Non-Clustered Index on Unique-key
             
			  
Composite Key :- composite key is a combination of two or more columns ,uniquely identify each row.

Foreign Key :-  FOREIGN KEY is a key used to link two tables together --> FOREIGN KEY is reference of primary key

Cluster index mysql :- PRIMARY KEY is cluster index --> only one cluster index
